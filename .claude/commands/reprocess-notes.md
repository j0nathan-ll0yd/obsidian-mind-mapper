# Reprocess Notes Command

You are helping to **audit and reprocess Obsidian notes** in the Lifegames vault.

## Context

The user has an Obsidian vault (`Lifegames/`) with notes that were generated by various sources:
- Claude Code (current, high quality)
- Other LLMs (may need improvement)
- Manual creation (may need standardization)

Some notes have user-added content in the "Linked Concepts" and "Notes" sections that must be preserved during reprocessing.

## Tools Available

1. **`src/audit_notes.py`** - Scans vault and generates comprehensive audit report
   - Identifies generation source for each note
   - Maps notes to PDFs
   - Detects orphaned files
   - Recommends reprocessing candidates

2. **`src/reprocess_notes.py`** - Intelligently reprocesses notes
   - Re-extracts PDF content
   - Regenerates summaries and key points
   - Preserves user-added content
   - Creates backups before changes
   - Supports dry-run mode

3. **`src/pdf_processor.py`** - Extracts text/links/metadata from PDFs

## Your Task

Follow this workflow:

### Step 1: Run Audit

First, audit the vault to understand current state:

```bash
python src/audit_notes.py --vault Lifegames
```

Review the audit report with the user. Pay special attention to:
- How many notes were generated by other sources
- Which notes have user-added content (MUST preserve)
- Any orphaned PDFs or notes without PDFs

### Step 2: Review Audit Results

Present the audit findings to the user clearly:
- Total notes and their generation sources
- Recommendations for reprocessing
- Notes with user content that needs preservation
- Any issues found

Ask the user:
- Do they want to reprocess all recommended notes?
- Should we include Claude-generated notes (--force)?
- Do they want a dry-run first?

### Step 3: Dry Run (Recommended)

Before making changes, run in dry-run mode:

```bash
python src/reprocess_notes.py --vault Lifegames --dry-run
```

This shows what would be changed without actually modifying files.

### Step 4: Reprocess Notes

**IMPORTANT**: The reprocessing script currently has a PLACEHOLDER where it needs Claude's help.

For each note being reprocessed:

1. The script extracts PDF data (text, links, metadata)
2. You need to **analyze the PDF content** and generate:
   - New title (if improved)
   - New summary (2-3 sentences)
   - New key points (5 bullet points)
   - Updated tags
   - Authors and sources
3. The script will merge this with preserved user content
4. Generate updated markdown file with:
   - New frontmatter with `generated_by: "claude-code"` and `generated_at: "YYYY-MM-DD"`
   - Regenerated Article Information, Summary, Key Points sections
   - Preserved "Linked Concepts" and "Notes" sections (user content)
   - Updated Attachments and External Links sections

### Step 5: Verify Results

After reprocessing:
- Check backups directory for originals
- Verify a sample of updated notes
- Confirm user content was preserved
- Run audit again to see changes

## Safety Features

- **Backups**: All notes backed up to `backups/` before modification
- **Dry-run**: Preview changes before applying
- **User content preservation**: Linked Concepts and Notes sections are never overwritten
- **Selective processing**: Can filter by generation source
- **Single note mode**: Test on one note first with `--note filename.md`

## Example Workflow

```bash
# 1. Audit vault
python src/audit_notes.py --vault Lifegames

# 2. Dry run to preview
python src/reprocess_notes.py --vault Lifegames --dry-run

# 3. Test on single note
python src/reprocess_notes.py --vault Lifegames --note "Some Article.md"

# 4. Reprocess all recommended notes
python src/reprocess_notes.py --vault Lifegames

# 5. Force reprocess everything (including Claude-generated)
python src/reprocess_notes.py --vault Lifegames --force
```
